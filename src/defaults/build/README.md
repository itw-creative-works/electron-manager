# `build/` — packaging assets

This directory holds files that go INTO the packaged app or are USED at packaging time. electron-builder auto-discovers some of these (`buildResources` defaults to `build/`).

## What goes here

| File | Purpose | Per-brand or universal? |
|---|---|---|
| `entitlements.mac.plist` | macOS hardened-runtime entitlements (network, files, JIT, etc.). Required for notarization. | Universal (rarely changes) |
| `icon.icns` / `icon.ico` / `icon.png` | App icons. electron-builder auto-discovers. | Per-brand |
| `installer.icns` (mac) | DMG/PKG installer icon. Optional. | Per-brand |
| `background.png` (mac) | DMG window background. Optional. | Per-brand |
| `installerIcon.ico` (win) | NSIS installer icon. Optional. | Per-brand |
| `certs/` | Code-signing certificates + provisioning profiles. **Per-brand. Never commit.** | Per-brand (gitignored) |

## Code-signing certificates

Cert files live in [`certs/`](certs/) and are **never committed** (gitignore handles `.p12`, `.cer`, `.mobileprovision`, `.p8`, `.pem`). Each developer / CI runner provides their own copies — the `.env` (also gitignored) just points env vars at them.

See [`certs/README.md`](certs/README.md) for the full inventory of files you need + which env vars reference each.

## What does NOT go here

- Source code → `src/`
- Build output → `dist/` (auto-generated)
- Released installers → `release/` (auto-generated by electron-builder)
- Runtime config → `config/`
